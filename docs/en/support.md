# Support and Theme Updates

This page explains the scope of technical support for the Esen theme, how to report an issue, how to prepare for an update, and where to find version information.

![Esen support and theme updates](/images/en/support.svg)

## Getting Support

If you need help, contact us through the official support channel provided by the platform where you purchased or downloaded the Esen theme. To help us investigate your issue efficiently, please include as much of the following information as possible:

- Your Shopify store URL;
- The version of Esen currently in use;
- A link to the affected page or template;
- A clear description of the issue and the expected result;
- Steps that reliably reproduce the issue;
- Your device, operating system, and browser version;
- Relevant screenshots or a screen recording;
- Any apps, app embeds, or custom code added before the issue appeared.

::: warning Account and Data Security
Never send your Shopify administrator password, customer personal information, payment information, API keys, or access tokens. If collaborator access is required for troubleshooting, grant only the minimum permissions needed through a secure official channel, and revoke access after the issue has been resolved.
:::

## Before Submitting an Issue

We recommend completing these checks before requesting technical support:

1. Confirm that the issue occurs in the Esen theme copy and page template you are currently using.
2. Save your changes in the theme editor, then test again in a private or incognito browser window.
3. Check whether the issue occurs on both desktop and mobile devices.
4. Duplicate the theme as an unpublished copy, then temporarily disable any recently added app embeds or custom code in that copy.
5. Confirm that the relevant products, pages, menus, and content are published to the correct sales channel.
6. Review the [Frequently Asked Questions](/en/faq) to see whether a solution is already available.

## Standard Support Coverage

Standard technical support for Esen generally includes:

- Guidance on theme installation and basic setup;
- Instructions for using Esen's built-in features in the theme editor;
- Explanations of features and settings covered by this documentation;
- Troubleshooting reproducible theme issues and compatibility problems;
- Workarounds or fix information for confirmed theme issues;
- Guidance on installing and performing routine updates to new Esen versions.

## Items Usually Outside Standard Support

The following items are generally outside the scope of standard Esen technical support:

- Installing, configuring, troubleshooting, or developing integrations for third-party apps;
- Developing custom pages, new features, or specific visual effects;
- Writing or debugging custom Liquid, CSS, or JavaScript;
- Issues caused by direct modifications to the theme source code;
- Questions about Shopify accounts, plans, payments, shipping, taxes, or domains;
- Store content entry, product data organization, or day-to-day store operations.

For these types of issues, we can help identify the likely source and may recommend contacting the relevant app developer, Shopify Support, or a qualified developer.

## Before Installing a Theme Update

Esen theme updates may include bug fixes, feature improvements, and compatibility changes. Before installing a future version:

1. Duplicate the currently published theme in Shopify admin so that you have a backup you can restore at any time.
2. Record the current theme version, installed apps, and custom modifications.
3. Back up custom Liquid, CSS, JavaScript, and any other files that were modified manually.
4. Install the new version as an unpublished theme instead of replacing the live theme directly.
5. Reconfigure theme settings, templates, app embeds, and custom code where necessary.
6. Test all critical store flows before publishing the new version, and keep the previous version temporarily in case you need to roll back.

::: warning Updates Do Not Automatically Merge Custom Content
A newly installed theme version is a separate theme copy. Theme settings, custom templates, app configurations, and code changes from the previous version may not migrate automatically. Review each item before publishing.
:::

## Post-Update Checklist

Before publishing an update, check at least the following:

- The home page and key marketing pages;
- Product details, variant selection, and add-to-cart behavior;
- Cart, search, and filtering features;
- Contact forms, newsletter forms, and other interactive components;
- Layout and behavior on desktop, tablet, and mobile devices;
- Language, market, and currency settings;
- Third-party app blocks and app embeds;
- Custom code and tracking pixels.

## Update and Compatibility Notes

- Updating the theme does not update the Shopify platform or third-party apps.
- Custom code and third-party app compatibility must be tested separately.
- Read the release notes for the relevant version before installing an update.
- Do not delete or overwrite the current live theme until the new version has been fully verified.
