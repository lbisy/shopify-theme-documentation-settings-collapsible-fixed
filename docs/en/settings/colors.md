# Colors

The **Colors** settings create reusable color schemes for backgrounds, text, borders, inputs, buttons, badges, and feedback messages. A scheme can be applied to any section that supports color schemes.

## Open the Colors settings

In Shopify admin, go to **Online Store > Themes**, click **Customize**, then open **Theme settings > Colors** and select the scheme you want to edit.

![Colors settings](/images/en/settings/colors.svg)

## Color schemes

Changing a scheme updates every section that uses it. You can create a light scheme for content, a dark scheme for footers or brand sections, and an accent scheme for promotions and calls to action.

Keep the number of schemes manageable and give each one a clear purpose. The panel preview is useful, but always confirm the final result on desktop and mobile.

## General colors

- **Background**: the section's base background color.
- **Background gradient**: replaces the solid background where supported.
- **Text color**: headings, body text, and some icons.
- **Border color**: dividers, card borders, and outlines.
- **Focus ring color**: the keyboard-focus outline around interactive elements.

Maintain strong contrast between text and its background. Focus rings must also remain easy to see.

::: warning Background gradients
When a gradient is set, supported areas use it instead of **Background**. Clear the gradient to restore the solid color.
:::

## Inputs

- **Input text**
- **Input background**
- **Input border**
- **Input placeholder**

Placeholder text should look different from entered text without becoming difficult to read. Test search, contact, and newsletter fields in default, focused, and completed states.

## Buttons

For the **Primary button**, set background, label, and border colors. Primary actions such as **Add to cart**, **Buy now**, and **Submit** should be prominent.

For the **Secondary button**, set background, label, and border colors. A transparent background creates an outline button; verify that both its text and border remain visible on every section background.

## Badges

Set the **Badge background**, **Badge label**, and **Badge border** colors used for labels such as Sale, New, or Sold out. Badges are small, so use clear contrast.

## Feedback messages

Each status has a background and foreground color:

- **Success**: completed actions.
- **Warning**: information requiring attention.
- **Error**: failed actions or invalid fields.
- **Info**: general notices.

::: info Accessibility
Do not communicate status through color alone. Keep the theme's text labels and icons so messages remain understandable to all visitors.
:::

## Remove a color scheme

Before selecting **Remove color scheme**, confirm that no section still depends on it and assign replacements where necessary. Removal may not be directly reversible, so record important color values first.

## Save and test

Click **Save**, then check text, links, buttons, inputs, badges, feedback messages, focus rings, and gradients on desktop and mobile. Hex values such as `#1B2900` can be copied between schemes to keep brand colors consistent.
