# 购物车与搜索

主题提供独立的购物车页面和搜索页面，并通过 **Cart Overlay**、**Search Overlay** 支持弹层式交互。本页说明相关页面模板设置，以及上线前需要验证的购物车和搜索流程。

![购物车与搜索设置](/images/zh/cart-search.svg)

## 打开页面模板

1. 进入 **Shopify 后台 → 在线商店 → 主题**。
2. 在当前主题上点击 **自定义**。
3. 使用编辑器顶部的页面选择器打开 **购物车** 或 **搜索**。
4. 在左侧 **模板** 分组中点击 **Cart** 或 **Search**。
5. 在右侧完成设置后点击 **保存**。

左侧的 **Overlay**、**Header** 和 **Footer** 是跨页面使用的分区或分区组；本页重点介绍 **模板** 下的 Cart 与 Search。全局购物车打开方式和预测搜索开关需要在 **主题设置** 中配置，详见[购物车设置](/zh/settings/cart)和[搜索设置](/zh/settings/search)。

## 购物车页面（Cart）

购物车页面用于显示已加入的商品、订单摘要和结账入口。空购物车左侧显示 **Cart** 标题和 **Shop More** 引导，右侧为 **Order Summary**；加入商品后，应进一步检查商品信息、数量调整和删除操作。

### 页面设置

- **Color scheme**：选择 Cart 分区的背景、文字、边框和按钮所使用的全局配色方案。
- **编辑方案**：进入全局颜色设置。修改方案会同时影响其他使用同一方案的分区，保存前请检查相关页面。
- **Heading size**：设置购物车主标题的字号，例如 **3XL**。
- **Padding Top**：调整 Cart 分区顶部留白，单位为 `px`，例如 `32 px`。
- **Padding Bottom**：调整 Cart 分区底部留白，单位为 `px`，例如 `32 px`。
- **自定义 CSS**：仅为当前分区添加样式覆盖。建议先使用主题提供的设置项；添加 CSS 后需同时检查桌面端和移动端。

选择配色时，应确保商品价格、数量控件、分隔线、折叠项和结账按钮都清晰可见。修改标题字号后，还要检查窄屏下标题是否异常换行，以及订单摘要是否被挤压。

### 订单摘要与配送估算

**Order Summary** 包含以下内容：

- **Order Special Instruction**：展开后供顾客填写订单备注或特殊要求。
- **Estimate Shipping**：选择国家/地区和省份，并填写邮政编码后获取运费估算。
- **Estimate Total**：显示当前预估总额。
- **Checkout**：进入 Shopify 结账流程；按钮旁的数字表示当前购物车商品数量。

运费估算结果取决于商店的配送区域、运费方案和顾客填写的地址。上线前应使用实际可配送和不可配送的地址分别测试；页面上的预估金额不等同于最终结账金额，税费、折扣和运费仍可能在结账时重新计算。

### 空购物车与有商品状态

配置时不要只检查空状态，还应分别验证：

- 空购物车能够显示明确提示和 **Shop More** 返回购物入口。
- 商品图片、标题、变体、单价和小计显示正确。
- 增加、减少数量后，商品小计、预估总额和购物车数量同步更新。
- 删除最后一件商品后，页面能够恢复为空购物车状态。
- 缺货、库存不足或数量受限时，页面提供清晰反馈。
- 订单备注可以输入、保存，并在进入结账流程后正确保留。
- **Checkout** 可以进入正常的 Shopify 结账页面。

::: tip 购物车打开方式
如果希望点击购物车图标后停留在当前页面并打开侧边面板，请在 **主题设置 → Cart → Cart type** 中选择 **Drawer**；如果希望进入本页，请选择 **Page**。
:::

## 搜索页面（Search）

搜索页面用于提交关键词并展示完整结果。结果类型包含 **PRODUCTS**、**ARTICLES** 和 **PAGES**；当关键词没有匹配内容时，页面会显示无结果提示。

### 页面设置

- **Color Scheme**：选择 Search 分区的背景、文字、输入框、标签和商品卡片所使用的全局配色方案。
- **编辑方案**：进入全局颜色设置。修改前请确认其他共用该方案的分区不会受到不符合预期的影响。
- **Search per page**：设置每页显示的搜索结果数量，例如 `12`；结果超过该数量时应显示分页。
- **Mobile product card columns (results and dropdown)**：设置移动端搜索结果和搜索下拉建议中的商品卡片列数，可选择 **1 column** 或 **2 columns**。
- **Padding Top**：调整 Search 分区顶部留白，单位为 `px`，例如 `32 px`。
- **Padding Bottom**：调整 Search 分区底部留白，单位为 `px`，例如 `32 px`。
- **模板设置**：展开后可查看该搜索模板提供的附加配置；实际项目中显示的选项以当前主题版本为准。
- **自定义 CSS**：仅为当前 Search 分区添加样式覆盖。修改后需检查搜索框、分类标签、结果网格和空状态。

每页结果数越大，顾客单次可以浏览的内容越多，但页面也会更长。移动端使用两列时信息更紧凑，适合图片和标题较短的商品；如果商品标题较长或卡片信息较多，可改为一列以提高可读性。

### 搜索结果与空状态

建议使用以下关键词组合测试完整搜索页：

1. 输入完整商品名称，确认目标商品排在合理位置。
2. 输入部分名称、品牌词或商品特征，检查相关结果。
3. 输入能够匹配博客文章或普通页面的关键词，确认 **ARTICLES** 和 **PAGES** 分类正常工作。
4. 切换结果类型，确认选中状态、下划线和结果内容同步变化。
5. 输入没有结果的关键词，确认提示中显示正确关键词且文案清晰。
6. 创建超过单页数量的结果，检查分页以及前后页跳转。

商品是否出现在搜索结果中会受到商品状态、销售渠道发布状态和 Shopify 搜索数据的影响。若刚修改商品标题或发布状态，可稍后刷新并再次测试。

### 预测搜索与 Search Overlay

**Search Overlay** 是点击页眉搜索入口后出现的搜索弹层，完整搜索页面则用于显示提交后的结果。可在左侧 **Overlay → Search Overlay** 中配置弹层，并在 **主题设置 → Search** 中开启或关闭 **Enable predictive search**。

开启预测搜索后，顾客输入关键词时可直接看到建议；关闭后仍可提交关键词并进入完整搜索页面。详细设置说明参见[搜索设置](/zh/settings/search)，弹层分区说明参见[搜索弹层](/zh/sections/search-overlay)。

## Cart Overlay 与 Search Overlay

两个 Overlay 分区与对应的完整页面相互独立：

- **Cart Overlay**：用于抽屉式购物车的内容和展示，可参见[购物车弹层](/zh/sections/cart-overlay)。
- **Search Overlay**：用于搜索输入和预测结果弹层，可参见[搜索弹层](/zh/sections/search-overlay)。
- **Newsletter Overlay**：属于独立的邮件订阅弹层，不控制购物车或搜索行为。

调整弹层后，应检查打开、关闭、遮罩点击、页面滚动和键盘焦点；移动端还要确认弹层没有被浏览器地址栏或屏幕安全区域遮挡。

## 配置示例

| 模板 | 设置项 | 示例值 |
| --- | --- | --- |
| Cart | Heading size | `3XL` |
| Cart | Padding Top / Bottom | `32 px / 32 px` |
| Search | Search per page | `12` |
| Search | Mobile product card columns | `2 columns` |
| Search | Padding Top / Bottom | `32 px / 32 px` |

配色方案的具体颜色取决于商店的全局颜色设置，因此没有固定色值。

## 发布前检查

- 分别测试空购物车、单件商品、多件商品、数量修改和删除商品。
- 检查订单备注、配送估算、预估总额和结账入口。
- 确认 Cart type 与预期一致，并测试页眉购物车图标和普通“加入购物车”按钮。
- 使用商品、文章、页面和无结果关键词测试搜索。
- 检查预测搜索、完整结果页、分类切换和分页。
- 确认移动端一列或两列商品卡片均没有文字重叠、图片异常裁切或触控区域过小。
- 使用键盘 `Tab`、`Shift + Tab`、方向键、`Enter` 和 `Esc` 检查搜索及弹层操作。
- 检查配色对比度、标题换行、上下间距，以及 Cart/Search 与页眉页脚的衔接。
- 最后点击 **保存**，并在桌面端和移动端各完成一次从搜索商品、加入购物车到进入结账页的完整流程。
