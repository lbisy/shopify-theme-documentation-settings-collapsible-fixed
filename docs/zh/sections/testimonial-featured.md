# 精选客户评价（Testimonial Featured）

Testimonial Featured（精选客户评价）分区用于以轮播形式展示顾客评价，并可搭配主视觉图片和悬浮信息卡。它适合放在首页、品牌故事页或商品介绍内容之间，用真实反馈帮助顾客了解使用体验。

![Testimonial Featured 设置面板与前台效果](/images/zh/testimonial-featured.svg)

## 添加分区

1. 进入 **Shopify 后台 → 在线商店 → 主题**。
2. 在当前主题上点击 **自定义**。
3. 打开需要编辑的页面或模板。
4. 点击 **添加分区**，选择 **Testimonial Featured**。
5. 上传主视觉图片，并按需配置悬浮卡片。
6. 点击 **添加 Testimonial**，填写客户评价。
7. 设置是否自动轮播及切换间隔。
8. 点击右上角的 **保存**。

## Color Scheme

选择当前分区使用的全局配色方案，影响背景、评价正文、评分、客户信息、悬浮卡片和轮播控件等元素。

点击 **编辑方案** 可进入全局颜色设置。修改方案本身会同时影响其他使用该方案的分区，因此应检查普通文字、星级图标、悬浮卡片和交互状态与背景之间是否具有足够的对比度。

## Image

选择与客户评价搭配的主视觉图片，也可以连接符合条件的动态来源。

适合使用产品使用场景、品牌氛围、细节或经过授权的客户图片。图片主体应放在安全区域，并为评价文字和悬浮卡片预留空间。上传后需分别检查桌面端和移动端的裁切。

若不需要图片，可以将该字段留空，并确认纯文字状态下的布局仍然完整。不要把评价正文只写入图片，以免影响多语言翻译和无障碍访问。

## Floating Card

### Show Floating Card

控制是否显示悬浮信息卡。该卡片可用于补充购买动态、内容类别、商品系列名称或其他简短提示。

关闭后，卡片标签和标题可以保留在编辑器中，但不会在该分区展示。

### Card Label

设置悬浮卡片中的辅助文字。该字段支持基础富文本格式、链接和动态来源。

建议使用简短类别或状态说明，避免文案过长导致卡片换行或遮挡主视觉内容。

### Card Title

设置悬浮卡片中的主要文字。该字段支持基础富文本格式、链接和动态来源。

主文字应与卡片标签形成清晰层级，并与当前图片或评价内容有关。启用悬浮卡片时，应同时检查两个字段；没有有效内容时建议关闭该功能。

::: tip 悬浮卡片位置
卡片可能覆盖在图片或分区边缘附近。修改图片、文字长度或页面语言后，都应重新检查卡片是否遮挡人物、商品或评价正文。
:::

## Slide Setting

### Auto Slide

控制多条客户评价是否自动切换。关闭后，顾客需要使用主题提供的轮播控件或手势切换内容。

只有一条 Testimonial 时，无需开启自动轮播。评价文字较长时，也可关闭自动切换，让顾客自行控制阅读节奏。

### Slide Delay

设置自动轮播时每条评价停留的时间，单位为秒。该设置仅在 **Auto Slide** 开启且存在多条评价时有意义。

切换间隔应足够顾客读完当前评价。短评价可以从 `5 s` 开始测试；文案较长时应延长间隔，避免内容在阅读完成前切换。

## Testimonial 内容块

点击 **添加 Testimonial** 可以增加一条客户评价。在左侧分区列表中可以拖动内容块调整顺序，也可以单独隐藏、复制或删除。

选择某个 Testimonial 后，根据内容块面板填写评价正文、客户姓名或署名、身份说明及评分等信息。具体字段名称可能因主题版本而有所不同。

### 评分

评分用于概括顾客满意度。应与评价原文和实际授权材料一致，不要为了统一视觉而修改顾客给出的评分。

如果所有评价都显示相同星级，应确认这是实际数据，而不是未修改的默认值。没有可靠评分时，可根据主题设置留空，而不是编造分数。

### 评价正文

保留顾客原意并控制长度，优先选择能够说明具体使用体验、服务过程或产品特点的内容。为了版面整齐可以进行必要的错别字修正或节选，但不应改变结论或夸大效果。

多条评价的长度应大致接近。文案差异过大会导致轮播高度变化，并可能造成页面内容跳动。

### 客户姓名与身份说明

姓名可以使用经授权的真实姓名、名字加姓氏首字母，或顾客同意的公开署名。身份说明可填写职业、使用场景、所在地区或客户类型，但不应披露不必要的个人信息。

没有获得公开身份信息的许可时，可以使用匿名化表达，并省略照片、公司或社交账号等可识别信息。

## Padding

- **Padding Top**：设置分区顶部与上一个分区之间的留白，单位为 `px`。
- **Padding Bottom**：设置分区底部与下一个分区之间的留白，单位为 `px`。

可先让上下间距保持一致，再根据图片高度、评价行数和悬浮卡片位置进行调整。移动端内容会纵向排列，应避免过大的留白拉长页面。

## 自定义 CSS

如果面板中提供 **自定义 CSS**，可以为当前分区添加局部样式，例如限制评价文字宽度或微调图片与悬浮卡片的位置。

自定义规则应限制在当前分区范围内。修改后需测试不同评价长度、自动切换、桌面端和移动端布局；主题升级后也应重新检查兼容性。

## 内容与合规建议

- 发布前取得顾客对评价文字、姓名、头像或其他身份信息的使用授权。
- 不得伪造评价、隐藏有重要影响的合作关系，或把付费推广描述为普通顾客反馈。
- 对原评价进行节选时保留原意，并在需要时使用明确的节选标识。
- 美容、健康、性能或收益类评价不能替代可验证证据，应符合销售地区的广告要求。
- 评价中不要包含订单号、联系方式、住址或其他敏感个人信息。
- 定期复核评价与当前商品、服务和政策是否仍然一致。

## 配置示例

| 设置项 | 示例值 |
| --- | --- |
| Color Scheme | 与页面背景有明显对比的方案 |
| Image | 产品使用场景图 |
| Show Floating Card | 开启 |
| Card Label | `顾客近期选择` |
| Card Title | `日常护理系列` |
| Auto Slide | 开启 |
| Slide Delay | `6 s` |
| Testimonial 1 | 评分、评价正文、客户署名和身份说明 |
| Testimonial 2 | 评分、评价正文、客户署名和身份说明 |
| Padding Top / Bottom | `32 px / 32 px` |

## 发布前检查

1. 确认主视觉图片清晰、裁切合理且拥有合法使用权。
2. 检查所有评价已取得发布授权，并移除敏感个人信息。
3. 核对评分、评价正文、客户署名和身份说明，没有保留默认内容或虚构信息。
4. 检查不同长度的评价切换时，分区高度不会明显跳动。
5. 开启悬浮卡片时，确认 Card Label 和 Card Title 清晰且不会遮挡关键内容。
6. 开启 Auto Slide 后，检查 Slide Delay 足够阅读完整评价。
7. 测试鼠标悬停、键盘聚焦或触摸操作时，自动轮播是否会妨碍阅读。
8. 使用轮播控件、键盘和触摸手势逐条切换评价，并确认当前状态容易识别。
9. 检查文字、星级、悬浮卡片及轮播控件与背景之间的对比度。
10. 分别在桌面端和移动端检查图片裁切、文字换行、卡片位置和上下间距。
11. 确认自定义 CSS 不会影响页面上的其他分区。
12. 点击 **保存**，再从实际店面完整浏览所有评价。
