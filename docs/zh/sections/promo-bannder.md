# 促销横幅（Promo Banner）

Promo Banner（促销横幅）分区用于组合主视觉图片、活动标题、跳转链接和圆形徽章，适合展示季节促销、新品系列、品牌活动或服务承诺。分区还支持添加 Card 内容块，用于补充与主横幅相关的信息。

![Promo Banner 设置面板与前台效果](/images/zh/promo-bannder.svg)

## 添加分区

1. 进入 **Shopify 后台 → 在线商店 → 主题**。
2. 在当前主题上点击 **自定义**。
3. 打开需要编辑的页面或模板。
4. 点击 **添加分区**，选择 **Promo Banner**。
5. 上传主视觉图片，填写标题并设置跳转目标。
6. 按需启用徽章或添加 Card 内容块。
7. 点击右上角的 **保存**。

## Color Scheme

选择当前分区使用的全局配色方案，影响背景、标题、辅助标题、链接、徽章和 Card 等元素。

点击 **编辑方案** 可进入全局颜色设置。修改方案本身会同时影响其他使用该方案的分区，因此应检查文字、链接和徽章与背景之间是否具有足够的对比度。

## Font

面板中的 **Font** 为字体相关设置区域。分区内文字会沿用主题的字体配置；如需调整全站字体，可前往 **主题设置 → Typography**。

## Heading

### Heading size

设置横幅主标题的字号。主标题是活动信息的视觉重点，建议使用简短且能够直接说明主题的文案。

字号较大时，应测试长标题在桌面端和移动端的换行，避免文字覆盖主视觉图片或占用过多纵向空间。

## Subtitle

### Subtitle size

设置辅助标题的字号。辅助标题通常用于标注活动类型、系列名称或简短引导，应与主标题形成清晰层级。

## Hero Banner

### Hero Image

选择横幅主视觉图片，也可以连接符合条件的动态来源。

建议使用清晰、高分辨率且与页面整体风格一致的图片。重要主体应放在安全区域内，并为标题、链接和徽章预留足够空间。图片可能根据容器尺寸发生裁切，上传后需分别检查桌面端与移动端。

若未选择图片，前台可能显示主题提供的占位内容，因此正式发布前应确认图片已正确加载。

### Subtitle

设置显示在主标题附近的辅助文案。该字段支持基础富文本格式、链接和动态来源。

可用于填写活动类别、适用范围或简短提示。不需要辅助信息时可以留空，避免保留无实际含义的默认内容。

### Heading

设置横幅主标题。该字段支持基础富文本格式、链接和动态来源。

建议突出一个核心信息，并把具体规则放在跳转后的活动页面中。标题不宜堆叠过多优惠条件，以免影响快速阅读。

### Link

设置横幅文字链接的跳转目标。可以选择店铺内的商品、商品系列、页面或其他有效地址，也可以使用搜索选择目标。该字段支持动态来源。

外部地址需使用完整网址。发布前应实际点击测试，确认目标页面可访问且与横幅内容一致。

### Link Text

设置链接在前台显示的文字，例如“查看活动”“浏览系列”或“了解详情”。该字段支持动态来源。

需要显示入口时，应同时填写 **Link** 和 **Link Text**；不需要跳转入口时，可将两个字段同时留空。

## Badge

### Show Badge

控制是否显示圆形徽章。徽章适合突出折扣幅度、服务承诺、限定信息或其他简短卖点。

### Badge Text

设置徽章外围显示的说明文字。该字段支持动态来源。

建议使用短语并控制字符数量。文字过长会使环形排版拥挤，在移动端也可能难以阅读。

### Badge Value

设置徽章中心显示的重点数值或短文本。该字段支持动态来源。

可填写百分比、数量或简短标识。徽章只负责展示信息，不会自动创建折扣或修改商品价格；相关优惠仍需在 Shopify 后台单独配置。

启用徽章时，应同时检查 **Badge Text** 与 **Badge Value**。如果没有有效内容，建议关闭 **Show Badge**。

## Card 内容块

点击 **添加 Card** 可在主横幅下添加补充内容。多个 Card 可用于介绍活动卖点、商品类别、服务保障或其他相关入口。

在左侧分区列表中可以拖动 Card 调整顺序，也可以单独隐藏、复制或删除。选择某个 Card 后，再填写该内容块提供的图片、文字或链接设置；不同主题版本中可用字段可能有所不同。

使用 Card 时建议：

- 每张 Card 只表达一个主题，并保持文案长度接近。
- 图片采用一致的比例、构图和视觉风格。
- 链接目标与 Card 文案准确对应。
- Card 数量较多时，重点检查移动端的排列、间距和页面长度。

## Padding

- **Padding Top**：设置分区顶部与上一个分区之间的留白，单位为 `px`。
- **Padding Bottom**：设置分区底部与下一个分区之间的留白，单位为 `px`。

可先让上下间距保持一致，再根据图片比例、标题换行及相邻分区背景进行调整。移动端空间有限，过大的留白会延长用户到达主要内容的距离。

## 自定义 CSS

展开 **自定义 CSS** 可以为当前分区添加局部样式。自定义规则应限制在当前分区范围内，避免影响页面上的其他横幅或全局组件。

修改前建议记录原始样式，并在桌面端、移动端以及主题编辑器预览中分别验证。主题升级后还应重新检查自定义规则是否仍然适用。

## 图片与内容建议

- 优先使用与横幅容器比例接近的图片，减少关键内容被裁切的风险。
- 不要把重要文案直接嵌入图片，以便多语言翻译、无障碍访问和移动端适配。
- 主标题说明核心活动，辅助标题补充范围，链接文字提示下一步操作。
- 图片、文字和徽章应保持足够对比度，不能只依赖颜色表达状态或含义。
- 促销金额、折扣比例、适用范围和结束时间应与实际活动规则一致。
- 为图片配置准确的替代文本，描述画面内容及其用途，避免只写“横幅图片”。

## 配置示例

| 设置项 | 示例值 |
| --- | --- |
| Color Scheme | 与页面背景有明显对比的方案 |
| Heading size | `XL` |
| Subtitle size | `XL` |
| Hero Image | 当季活动主视觉图 |
| Subtitle | `夏季精选` |
| Heading | `会员限时礼遇` |
| Link | 活动商品系列页面 |
| Link Text | `查看活动` |
| Show Badge | 开启 |
| Badge Text | `限时会员优惠` |
| Badge Value | `20%` |
| Padding Top / Bottom | `32 px / 32 px` |

## 发布前检查

1. 确认 Hero Image 已上传，清晰度足够且拥有合法使用权。
2. 分别在桌面端和移动端检查图片裁切，确保主体没有被标题、链接或徽章遮挡。
3. 检查 Heading 与 Subtitle 的层级、换行和对比度，并移除无意义的默认内容。
4. 实际点击文字链接，确认 Link 与 Link Text 均已填写且目标正确。
5. 启用徽章时，检查 Badge Text 的环形排版和 Badge Value 的可读性。
6. 确认徽章中的折扣或承诺与商品价格、折扣设置和活动条款一致。
7. 检查所有 Card 的内容顺序、图片比例、文字长度和链接目标。
8. 使用键盘 `Tab` 和 `Enter` 测试横幅及 Card 中的链接，确认焦点状态清晰可见。
9. 检查上下间距以及该分区与页眉、前后分区的衔接。
10. 点击 **保存**，再从实际店面完成一次横幅浏览与跳转流程。
