# 自定义 Liquid（Custom Liquid）

Custom Liquid（自定义 Liquid）分区用于在主题编辑器中加入少量自定义 HTML、Liquid 输出或现有主题代码片段。它适合展示主题原有分区无法完成的简单动态内容，但不应替代完整的主题开发或应用功能。

![Custom Liquid 设置面板与前台效果](/images/zh/custom-liquid.svg)

::: warning 修改前备份
Liquid 代码错误可能导致当前分区无法显示，也可能破坏页面布局。修改前建议复制当前主题，并保存一份可正常工作的代码。
:::

## 添加分区

1. 进入 **Shopify 后台 → 在线商店 → 主题**。
2. 在当前主题上点击 **自定义**。
3. 打开需要使用自定义内容的页面或模板。
4. 点击 **添加分区**，选择 **Custom Liquid**。
5. 选择分区宽度，并在 **Custom liquid** 编辑框中输入代码。
6. 调整配色和上下间距。
7. 在桌面端与移动端完成测试后，点击右上角的 **保存**。

面板顶部的 **Learn how to use this section** 可打开该分区的使用帮助。

## Color scheme

选择当前分区使用的全局配色方案，控制分区背景和主题样式能够识别的文字、链接等元素颜色。

点击 **编辑方案** 可进入全局颜色设置。修改方案本身会同时影响其他使用该方案的分区。

自定义代码中如果写入固定颜色，可能不会自动跟随当前配色方案。建议优先复用主题现有 CSS 类或颜色变量，并分别检查浅色和深色方案。

## Section width

设置自定义内容的横向宽度。

- **Fixed width**：内容限制在主题的页面容器内，左右通常保留统一边距，适合文字、表格和普通嵌入内容。
- **Full width**：内容扩展到可用页面宽度，适合通栏背景、横幅或需要贴近视口边缘的布局。

全宽只决定外层分区宽度。自定义代码内部如果设置了固定宽度、最大宽度或额外边距，实际内容仍可能无法铺满页面。

## Custom liquid

在该编辑框中输入 HTML 和 Shopify Liquid。Liquid 会在店面页面生成时执行，再把结果输出为 HTML。

常用语法包括：

- <code v-pre>{{ ... }}</code>：输出对象属性或表达式结果。
- `{% ... %}`：执行条件、循环、变量赋值或渲染片段等逻辑。
- `| filter`：对输出内容进行格式化、转义或转换。
- HTML 标签：定义最终显示的内容结构。

### 示例：添加简单文字

```liquid
<div class="rte">
  <p>这里填写需要展示的补充说明。</p>
</div>
```

`rte` 是许多 Shopify 主题用于富文本内容的通用类。若当前主题没有对应样式，应改用主题实际提供的类名。

### 示例：仅在商品页面显示供应商

```liquid
{% if product %}
  <p>品牌：{{ product.vendor | escape }}</p>
{% endif %}
```

条件判断可以避免在没有 `product` 对象的页面上输出无效内容。`escape` 过滤器会转义 HTML 特殊字符，适合处理最终要作为普通文字显示的数据。

### 示例：调用主题中已有的 snippet

```liquid
{% render 'snippet-name' %}
```

必须先确认主题的 `snippets` 目录中存在同名文件。名称错误、文件被删除或 snippet 依赖当前页面不存在的对象时，内容可能无法正常渲染。

## 页面上下文

Custom Liquid 可使用的对象取决于分区所在模板：

- 商品模板通常可以访问 `product`。
- 商品系列模板通常可以访问 `collection`。
- 博客文章模板通常可以访问 `article` 和 `blog`。
- 普通页面模板通常可以访问 `page`。

同一段代码移动到其他模板后，对象可能为空。因此，引用模板专属对象时应先使用条件判断，并在所有实际使用该分区的模板中测试。

## 适合使用的内容

- 少量静态 HTML 或补充说明。
- 基于当前商品、商品系列、文章或页面对象的简单内容。
- 调用主题中已经存在且经过测试的 snippet。
- 简单条件判断、循环和格式化输出。
- 应用开发者明确要求放入 Custom Liquid 的兼容代码。

## 不适合直接放入的内容

- 完整的新分区架构或大量复杂业务逻辑。
- `{% schema %}` 设置定义；自定义编辑框不能借此创建新的主题编辑器设置或内容块。
- 来源不明的脚本、追踪代码或经过混淆的代码。
- 需要密钥、访问令牌或其他机密信息的代码。Liquid 和前端源码可能被顾客查看，不能用于保存秘密。
- 大量重复 HTML、CSS 或 JavaScript；这类内容更适合放入主题文件并进行版本管理。
- 应用已经提供 App block 或 App embed 的功能；此时优先使用应用的正式集成方式。

## 输出安全

当商品标题、元字段、顾客输入或其他可变数据需要作为普通文字输出时，应根据上下文使用合适的转义过滤器，例如：

```liquid
{{ product.title | escape }}
```

不要把不可信内容当作可执行 HTML 输出，也不要将动态数据未经处理地拼接进 HTML 属性、URL、CSS 或 JavaScript。

自定义代码可能加载第三方资源或收集数据。添加之前应确认来源、性能影响、隐私政策和顾客同意要求。

## Padding

- **Padding Top**：设置分区顶部与上一个分区之间的留白，单位为 `px`。
- **Padding Bottom**：设置分区底部与下一个分区之间的留白，单位为 `px`。

自定义 HTML 自身也可能包含 `margin` 或 `padding`。如果页面间距异常，应同时检查分区设置与代码内部样式，避免重复留白。

## 自定义 CSS

展开 **自定义 CSS** 可以为当前分区添加局部样式。需要少量样式调整时，应优先使用该区域，而不是在 **Custom liquid** 中反复加入 `<style>` 标签。

为自定义内容添加唯一类名，并把 CSS 限制在该类名下，能够降低样式影响其他分区的风险。例如：

```html
<div class="custom-note">
  <p>补充说明</p>
</div>
```

```css
.custom-note {
  max-width: 48rem;
  margin-inline: auto;
}
```

主题升级后应重新检查自定义类、变量和 snippet 是否仍然存在。

## 常见问题

### 保存后没有内容

- 检查 `if` 条件是否始终为假。
- 确认当前模板提供代码引用的 Liquid 对象。
- 检查输出标签 <code v-pre>{{ }}</code> 与逻辑标签 `{% %}` 是否混用。
- 确认调用的 snippet 文件名称正确。
- 暂时换成一段简单 HTML，判断问题来自分区还是 Liquid 逻辑。

### 页面布局异常

- 检查是否缺少 HTML 闭合标签。
- 删除固定宽度、负边距或绝对定位后重新测试。
- 检查 Full width 与代码内部容器宽度是否冲突。
- 暂时移除自定义 CSS 和第三方嵌入代码，逐段恢复定位问题。

### 某些页面正常，其他页面为空

通常是代码依赖特定模板对象。应为 `product`、`collection`、`article` 或 `page` 等对象增加存在性判断，或者只把该分区添加到相应模板。

### 编辑器预览与实际店面不同

保存后在实际店面重新测试，并检查不同市场、语言、登录状态和设备宽度。第三方脚本还可能受隐私同意、浏览器限制或加载顺序影响。

## 发布前检查

1. 确认已备份主题和当前可用代码。
2. 检查 Liquid 标签、HTML 标签和引号是否完整闭合。
3. 确认模板专属对象都有存在性判断。
4. 对作为普通文字显示的动态数据使用合适的转义过滤器。
5. 检查代码不包含密钥、令牌、个人信息或来源不明的脚本。
6. 分别测试 Fixed width 与 Full width，并检查桌面端和移动端布局。
7. 检查浅色和深色配色下的文字、链接及焦点状态。
8. 使用键盘测试自定义内容中的所有链接、按钮和表单控件。
9. 检查上下间距与代码内部 `margin`、`padding` 没有重复。
10. 测试不同语言、市场及该模板可能使用的不同资源数据。
11. 确认第三方嵌入符合隐私要求，且不会明显影响页面性能。
12. 点击 **保存**，再从实际店面完成完整浏览和交互测试。
