# 购物车弹层（Cart Overlay）

购物车弹层用于在顾客点击购物车入口或将商品加入购物车后，以侧边抽屉的形式显示购物车内容。弹层打开时，顾客仍停留在当前页面；页面主体会被遮罩覆盖，方便顾客快速确认商品、调整购物车并进入结账。

![购物车弹层设置面板与前台效果](/images/zh/cart-overlay.svg)

## 打开设置

1. 进入 **Shopify 后台 → 在线商店 → 主题**。
2. 在当前主题上点击 **自定义**。
3. 在主题编辑器左侧找到 **Overlay** 分组。
4. 点击 **Cart Overlay**。
5. 完成设置后，点击右上角的 **保存**。

Cart Overlay 是 Overlay 分组中的弹层分区，不需要添加到某个页面模板中。它可跨页面使用；预览时可打开商品页并点击购物车入口，或将商品加入购物车，以检查实际效果。

::: info 启用抽屉式购物车
如果点击购物车图标后进入独立购物车页面，请前往 **主题设置 → Cart**，将 **Cart type** 设为 **Drawer**。Cart Overlay 负责抽屉的外观和内容，Cart type 决定顾客打开购物车时使用抽屉还是独立页面。
:::

## 设置项

### Drawer position

设置购物车抽屉从屏幕哪一侧打开：

- **Left**：抽屉从左侧滑出。
- **Right**：抽屉从右侧滑出。

建议根据页面布局和主要导航位置选择方向。保存后应分别在桌面端和移动端测试，确认抽屉不会遮挡重要操作，关闭按钮也容易点击。

### Color scheme

选择购物车弹层使用的全局配色方案。该方案会影响抽屉背景、标题、正文、分隔线、数量控件以及底部操作区域等元素的颜色。

点击 **编辑方案** 可进入全局颜色设置。修改配色方案本身时，其他使用同一方案的分区也会同步变化；保存前请确认文字、边框和按钮与背景之间具有足够的对比度。

### 自定义 CSS

展开 **自定义 CSS** 可为当前 Cart Overlay 添加样式覆盖。建议优先使用主题自带的方向和配色设置；添加 CSS 后，需要同时检查空购物车、有商品、桌面端和移动端四种状态。

## 前台显示内容

购物车弹层主要包括：

- 顶部的 **Cart** 标题与关闭按钮。
- 中部的购物车商品列表；空购物车时显示 **Your cart is empty.**。
- 底部的 **Total**（合计金额）。
- **Clear all**（清空购物车）按钮，括号中的数字表示当前购物车商品数量。
- **Checkout**（结账）按钮，括号中的数字表示当前购物车商品数量。

当购物车为空时，清空和结账按钮可能显示为不可用状态；加入商品后，应确认商品信息、金额和按钮状态会同步更新。实际文案和货币格式取决于店铺语言、市场及货币设置。

## 删除分区

面板底部的 **删除分区** 会从当前主题结构中移除 Cart Overlay。删除后，抽屉式购物车可能无法正常显示，因此只有在确定不使用购物车弹层时才应执行此操作。若只是希望改用独立购物车页面，通常只需在 **主题设置 → Cart → Cart type** 中选择 **Page**。

## 使用建议

- 抽屉打开时，确认页面遮罩完整显示，背景页面不会被误操作。
- 测试右上角关闭按钮、遮罩点击和 `Esc` 键是否能正常关闭弹层。
- 分别测试空购物车、单件商品、多件商品以及删除最后一件商品后的状态。
- 修改商品数量后，确认商品小计、**Total** 和按钮旁的数量同步更新。
- 点击 **Clear all** 前确认清空操作符合预期；点击 **Checkout** 应能进入正常的 Shopify 结账流程。
- 使用键盘 `Tab` 和 `Shift + Tab` 检查焦点顺序，确保焦点不会进入被遮罩的页面内容。
- 在移动端检查抽屉宽度、长商品标题、价格、数量控件和底部按钮，避免内容被屏幕边缘或安全区域遮挡。

## 发布前检查

1. 在商品页将商品加入购物车，确认 Cart Overlay 按预期方向打开。
2. 从页眉购物车图标再次打开弹层，确认行为一致。
3. 增加、减少和删除商品，检查数量与总金额。
4. 清空购物车，确认显示正确的空状态。
5. 检查浅色与深色配色下的文字、边框和按钮对比度。
6. 分别在桌面端和移动端完成一次从加购到进入结账页的完整流程。
