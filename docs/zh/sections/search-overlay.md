# 搜索弹层（Search Overlay）

Search Overlay（搜索弹层）是点击页眉搜索入口后打开的抽屉式搜索界面。顾客可以在不离开当前页面的情况下输入关键词、查看预测建议，并进一步进入完整搜索结果页。

![Search Overlay 设置面板与前台效果](/images/zh/search-overlay.svg)

## 与完整搜索页面的区别

- **Search Overlay**：覆盖在当前页面上方，用于快速输入关键词和显示有限数量的预测结果。
- **Search 页面**：提交搜索后打开的完整页面，用于展示全部结果、结果分类和分页。

两者相互配合，但设置彼此独立。弹层结果数量不会改变完整搜索页面的每页结果数量。

## 打开设置

1. 进入 **Shopify 后台 → 在线商店 → 主题**。
2. 在当前主题上点击 **自定义**。
3. 在左侧找到 **Overlay** 分组。
4. 点击 **Search Overlay**。
5. 配置配色、抽屉位置和预测结果数量。
6. 点击右上角的 **保存**。

Search Overlay 属于跨页面使用的 Overlay 分组，而不是只属于当前模板的普通内容分区。修改后应在首页、商品页、商品系列页和其他带有页眉搜索入口的页面上测试。

如果左侧没有该分区，可在 Overlay 分组中点击 **添加分区**，重新添加 **Search Overlay**。

## Color scheme

选择搜索弹层使用的全局配色方案，影响抽屉背景、标题、输入框、关闭按钮、预测结果和分隔线等元素。

点击 **编辑方案** 可进入全局颜色设置。修改方案本身会同时影响其他使用该方案的分区，因此应检查：

- 输入文字与占位提示是否容易区分。
- 结果标题、价格和辅助信息是否清晰。
- 关闭按钮、链接悬停和键盘焦点状态是否可见。
- 抽屉与页面遮罩之间是否具有明确边界。

## Drawer position

设置搜索抽屉从页面哪一侧打开：

- **Left**：抽屉显示在页面左侧。
- **Right**：抽屉显示在页面右侧。

可根据页眉搜索图标位置、阅读方向及其他抽屉组件进行选择。搜索弹层与购物车弹层使用不同侧边时，更容易区分；使用同一侧边时，则可保持全站交互一致。

多语言店铺还应检查从右到左语言的布局，确保输入方向、关闭按钮和抽屉位置符合阅读习惯。

## Predictive search result limit

设置预测搜索区域最多显示的结果数量。顾客输入关键词时，实际结果可能少于该值，具体取决于可搜索内容、关键词相关性、主题设置和 Search & Discovery 配置。

数值较小可以让弹层保持简洁并减少纵向滚动；数值较大能够展示更多入口，但会增加移动端浏览负担。可以从 `8` 条开始测试，再根据商品数量和结果类型调整。

该设置只控制预测区域的显示上限，不改变：

- 完整搜索页面的每页结果数量。
- 商品、文章或页面是否可被搜索。
- Shopify 搜索结果的相关性排序。
- Search & Discovery 中配置的结果类型、同义词或商品提升规则。

## 启用预测搜索

Search Overlay 的数量设置不等同于预测搜索总开关。请前往 **主题设置 → Search**，确认 **Enable predictive search** 已开启。

- 开启后，顾客输入内容时可以看到预测建议。
- 关闭后，弹层仍可用于输入和提交关键词，但不会在输入过程中显示预测结果。

相关全局设置可参见[搜索设置](/zh/settings/search)，完整搜索页说明可参见[购物车与搜索](/zh/cart-search)。

## Search & Discovery

Shopify Search & Discovery 可用于调整搜索和预测搜索的结果类型、商品提升、同义词及缺货商品处理方式。

预测搜索可能返回商品、商品系列、页面、博客文章和建议查询，实际类型取决于主题请求及应用设置。如果应用中的调整没有反映到店面，可能是主题在请求中固定了结果类型，此时需要联系主题开发者核对实现。

优化搜索内容时建议：

- 使用清晰且一致的商品标题、商品类型、供应商和变体名称。
- 为常见同义词、简称和不同拼写建立同义词组。
- 只提升与目标关键词真正相关的商品。
- 检查缺货商品是显示、隐藏还是排列在结果末尾。
- 确保页面和博客文章使用能够被顾客理解的标题。

## 模板设置

展开 **模板设置** 可以查看当前主题版本为搜索弹层提供的附加选项。不同版本中可用内容可能不同，应以面板实际显示为准。

修改后需同时测试预测结果弹层和完整搜索页面，避免模板级设置与 Search & Discovery 配置产生不一致。

## 自定义 CSS

展开 **自定义 CSS** 可以为当前搜索弹层添加局部样式，例如微调抽屉宽度、结果间距或分隔线。

自定义规则应限制在 Search Overlay 范围内，避免影响完整搜索页面、页眉搜索框或其他抽屉。修改后应检查桌面端、移动端、空状态、加载状态、长关键词和多种结果类型；主题升级后也需重新验证。

## 删除分区

面板底部的 **删除分区** 会从 Overlay 分组移除 Search Overlay。删除后，页眉搜索入口可能无法再打开抽屉，或改用主题提供的其他搜索行为。

除非确定不再使用弹层搜索，否则建议先隐藏或在主题副本中测试。需要恢复时，可通过 Overlay 分组的 **添加分区** 重新添加。

## 搜索内容与可见性

商品、页面和博客文章能否出现在搜索结果中，取决于其发布及可搜索状态。某项内容没有出现时，请检查：

- 商品是否为有效状态并发布到在线商店。
- 商品是否被设置为不在 Shopify 搜索中显示的状态。
- 页面或博客文章是否已发布且标题完整。
- 关键词是否出现在 Shopify 支持搜索的属性中。
- Search & Discovery 是否限制了该结果类型。
- 内容刚修改时，搜索索引是否尚未更新。

不要通过增加 Predictive search result limit 解决内容不可搜索的问题；应先修正内容状态和搜索数据。

## 无障碍与交互要求

打开搜索弹层后，应满足以下交互预期：

- 键盘焦点进入搜索输入框或抽屉中的首个可操作元素。
- `Tab` 和 `Shift+Tab` 能按合理顺序访问输入框、结果和关闭按钮。
- 方向键、`Enter` 等操作能够选择并打开预测结果。
- `Esc`、关闭按钮及主题支持的遮罩点击能够关闭弹层。
- 关闭后，焦点返回原来的页眉搜索按钮。
- 弹层打开时，背景页面不会被误操作或发生不受控滚动。
- 输入中、加载、无结果和错误状态都有清晰反馈。

移动端还应确认抽屉不被浏览器地址栏、软键盘或屏幕安全区域遮挡。

## 配置示例

| 设置项 | 示例值 |
| --- | --- |
| Color scheme | 浅色高对比方案 |
| Drawer position | `Right` |
| Predictive search result limit | `8` |
| Enable predictive search | 开启 |
| Search & Discovery | 启用商品、页面与博客文章结果 |

## 发布前检查

1. 从首页、商品页、商品系列页和博客页分别打开 Search Overlay。
2. 确认抽屉从指定侧边打开，并且桌面端与移动端位置一致。
3. 使用完整商品名、部分商品名、供应商、文章标题和页面标题测试搜索。
4. 输入无结果关键词、特殊字符、中文与英文混合内容及较长关键词。
5. 核对预测结果数量不会超过设置值，并测试实际结果少于上限的状态。
6. 点击商品、商品系列、文章、页面和建议查询，确认目标正确。
7. 按 `Enter` 提交关键词，确认完整搜索页面显示相同查询及合理结果。
8. 测试 Search & Discovery 中的同义词、商品提升、结果类型和缺货商品设置。
9. 使用键盘完成打开、输入、选择结果和关闭流程，并确认焦点返回搜索入口。
10. 在移动端测试软键盘弹出、抽屉滚动、结果触控区域和关闭操作。
11. 检查输入框、结果文字、价格、分隔线、关闭按钮和焦点状态的对比度。
12. 确认自定义 CSS 不会影响完整搜索页面或其他弹层。
13. 点击 **保存**，再从实际店面完成一次预测搜索和完整搜索流程。
