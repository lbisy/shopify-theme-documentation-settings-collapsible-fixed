# 促销倒计时（Promotion Countdown）

Promotion Countdown（促销倒计时）分区用于展示限时活动的剩余时间。分区可包含辅助标题、主标题、天数、小时、分钟、秒以及行动按钮，适合新品发布、季节促销、会员活动和预约提醒等场景。

![Promotion Countdown 设置面板与前台效果](/images/zh/promotion-countdown.svg)

## 添加分区

1. 进入 **Shopify 后台 → 在线商店 → 主题**。
2. 在当前主题上点击 **自定义**。
3. 打开需要编辑的页面或模板。
4. 点击 **添加分区**，选择 **Promotion Countdown**。
5. 设置活动文案和结束日期，并按需配置按钮。
6. 点击右上角的 **保存**。

## Color Scheme

选择当前分区使用的全局配色方案，影响背景、文字、倒计时数字、分隔符和按钮等元素。

点击 **编辑方案** 可进入全局颜色设置。修改方案本身会同时影响其他使用该方案的分区，因此应检查背景与文字、数字卡片及链接之间是否具有足够的对比度。

## Font

面板中的 **Font** 为字体相关设置区域。分区内文字会沿用主题的字体配置；如需修改全站字体，可前往 **主题设置 → Typography**。

## Heading

### Heading size

设置主标题字号。主标题通常是分区中视觉层级最高的活动文案；字号越大，越需要检查长文案在移动端的换行效果。

### Heading

设置活动主标题。该字段支持基础富文本格式、链接和动态来源。

建议用一句简短文案说明优惠、活动主题或关键利益点。没有主标题时可以留空，避免保留无实际含义的默认内容。

## Subtitle

### Subtitle size

设置辅助标题字号。辅助标题应与主标题形成明确的视觉层级，不宜同时使用过大的字号。

### Sub Heading

设置显示在主标题附近的辅助文案。该字段支持基础富文本格式、链接和动态来源。

可用于填写活动阶段、会员范围或简短提示；不需要时可以留空。文字较长时，应同时检查桌面端与移动端的行数和间距。

## End Date

设置倒计时的目标日期和时间，格式必须为：

```text
YYYY-MM-DD HH:MM:SS
```

例如，`2027-01-15 20:00:00` 表示目标日期为 2027 年 1 月 15 日 20:00:00。月份、日期、小时、分钟和秒均应使用两位数字，并使用半角连字符、冒号和空格。

::: warning 日期与时间
保存后应在实际店面确认倒计时结果，并使用活动主要市场所在时区进行校对。若时间已过、格式不正确或字段为空，倒计时可能无法按预期显示。
:::

## 时间单位标签

- **Days Label**：设置“天”对应的标签。
- **Hours Label**：设置“小时”对应的标签。
- **Minutes Label**：设置“分钟”对应的标签。
- **Seconds Label**：设置“秒”对应的标签。

这些字段只更改单位名称，不更改倒计时的计算方式。多语言店铺应为每种语言提供对应翻译，并保持四个标签的语言和大小写风格一致。标签不宜过长，以免在窄屏设备上挤压数字区域。

## CTA 按钮

### Show CTA Button

控制是否显示行动按钮。关闭后，按钮文字和链接仍可保留在编辑器中，但不会在该分区展示。

### CTA Text

设置按钮上显示的文字。建议使用明确的动作表达，例如“查看活动”“立即选购”或“设置提醒”。该字段支持动态来源。

### CTA Link

设置点击按钮后打开的目标。可以选择店铺内的商品、商品系列、页面或其他有效地址，也可以使用搜索选择目标。该字段支持动态来源。

启用按钮时，应同时填写按钮文字与链接。外部地址需使用完整网址；发布前应确认目标页面可访问，并与活动内容一致。

## Padding

- **Padding Top**：设置分区顶部与上一个分区之间的留白，单位为 `px`。
- **Padding Bottom**：设置分区底部与下一个分区之间的留白，单位为 `px`。

可先让上下间距保持一致，再根据相邻分区的背景、标题换行和页面节奏进行调整。移动端空间有限，过大的间距会使倒计时主体离开首屏。

## 前台显示与内容建议

倒计时按天、小时、分钟和秒显示剩余时间。天数可能超过两位，而小时、分钟和秒通常以两位数字显示；设置配色和字号时，应为较长的天数预留空间。

- 将结束时间与实际活动规则、折扣生效时间和落地页信息保持一致。
- 主标题优先说明活动价值，辅助标题补充范围或条件，避免重复表达。
- 时间单位使用顾客熟悉的语言，不要混用多种语言或缩写风格。
- 按钮应指向可继续完成操作的页面，避免链接到无关内容或已结束活动。
- 不要仅依赖倒计时传达重要条件；库存、地区限制和优惠规则应在活动页面中清楚说明。

## 配置示例

| 设置项 | 示例值 |
| --- | --- |
| Color Scheme | 与页面背景有明显对比的方案 |
| Heading size | `2XL` |
| Heading | `会员专享活动即将结束` |
| Subtitle size | `XL` |
| Sub Heading | `限时优惠` |
| End Date | `2027-01-15 20:00:00` |
| Days / Hours / Minutes / Seconds Label | `天 / 小时 / 分钟 / 秒` |
| Show CTA Button | 开启 |
| CTA Text | `查看活动` |
| CTA Link | 活动商品系列页面 |
| Padding Top / Bottom | `32 px / 32 px` |

## 发布前检查

1. 确认 **End Date** 使用规定格式，且目标时间晚于发布时间。
2. 在实际店面核对剩余天数和时间，确认活动时区没有偏差。
3. 检查四个时间单位的顺序、翻译和排版，并测试天数较长时的显示效果。
4. 确认主标题与辅助标题没有无意义的默认内容，长文案在移动端不会溢出。
5. 开启按钮时，确认 **CTA Text** 和 **CTA Link** 均已填写，并实际点击测试。
6. 检查浅色和深色配色下的文字、数字卡片、分隔符及按钮对比度。
7. 分别在桌面端和移动端检查数字排列、标签间距与上下留白。
8. 使用键盘 `Tab` 和 `Enter` 测试按钮，确认焦点状态清晰可见。
9. 预先测试倒计时结束后的页面状态，并准备及时更新或移除活动内容。
10. 点击 **保存**，再从实际店面完成一次活动浏览流程。
