# 商品对比表（Comparison Table）

Comparison Table（商品对比表）分区用于并列展示多件商品的图片、名称、价格、评分、自定义属性和库存状态，帮助顾客比较不同型号或同类商品。它适合放在商品模板、商品系列落地页或选购指南页面中。

![Comparison Table 设置面板与前台效果](/images/zh/product-comparison-table.svg)

## 使用前准备

1. 确认需要比较的商品已创建并发布到在线商店。
2. 为商品设置清晰的主图、准确的标题、价格和库存信息。
3. 如需显示评分，确认商品评分数据已由兼容的评论应用或商品元字段提供。
4. 如需比较成分、材质、尺寸等自定义属性，先为所有商品建立相同定义和类型的商品元字段。
5. 为每件商品填写对应字段值，再进入主题编辑器配置比较表。

对比项越统一，顾客越容易理解差异。不要使用含义相同但名称、单位或格式不同的商品数据。

## 添加分区

1. 进入 **Shopify 后台 → 在线商店 → 主题**。
2. 在当前主题上点击 **自定义**。
3. 打开需要编辑的商品模板或其他页面模板。
4. 点击 **添加分区**，选择 **Comparison Table**。
5. 在 **Select Products** 中选择需要比较的商品。
6. 添加并排列需要显示的比较行。
7. 配置突出列、颜色和上下间距。
8. 点击右上角的 **保存**。

## Color scheme

选择当前分区使用的全局配色方案，影响表格背景、标题、商品信息、边框和比较行文字等元素。

点击 **编辑方案** 可进入全局颜色设置。修改方案本身会同时影响其他使用该方案的分区，因此应检查表头、正文、价格和边框与背景之间是否具有足够的对比度。

## Section heading

设置对比表上方的分区标题。该字段支持基础富文本格式、链接和动态来源。

标题应说明比较对象或使用场景，例如“同系列商品对比”或“选择适合你的型号”。不需要标题时可以留空，避免保留无实际意义的默认内容。

## Heading

### Heading size

设置分区标题的字号。较大的字号可以强化页面层级，但长标题可能在移动端换行，应与表格宽度和上方内容协调。

### Highlight column background

设置突出商品列的背景颜色。该颜色只用于强调指定列，不会改变商品本身的状态或排序。

突出背景应与普通列形成可辨识的差异，同时保证商品标题、价格、评分和库存文字仍然清晰。不要只依赖背景颜色传达“推荐”等含义；如需表达推荐理由，应在页面文案中补充说明。

### Availability in stock color

设置有库存状态文字的颜色。建议选择在当前背景上对比度足够、且符合店铺视觉规范的颜色。

### Availability out of stock color

设置缺货状态文字的颜色。该颜色用于视觉提示，不会改变 Shopify 中的库存数量或商品是否允许继续销售。

有库存与缺货状态除了颜色外还应保留清晰文字，使色觉差异用户也能理解当前状态。

## Select Products

选择需要显示在对比表中的商品。已选商品会按照列表中的顺序形成各个商品列；使用 **更改** 可以重新选择或调整商品集合。

选择商品时建议：

- 比较用途、品类或价格区间相近的商品。
- 控制商品数量，避免表格过宽并增加移动端浏览负担。
- 确保每件商品都有主图和简洁、可区分的标题。
- 检查商品是否在顾客当前市场可用，价格和币种是否正确。
- 更换商品后重新核对突出列索引和所有自定义字段值。

商品较多时，桌面端可能只显示部分列，移动端通常需要横向浏览。应确认顾客能够发现并顺畅操作横向区域。

## Highlight Column Index

设置需要突出显示的商品列。索引按 **Select Products** 中的商品顺序计算：

- `0`：不突出任何商品列。
- `1`：突出第一个商品列。
- `2`：突出第二个商品列，后续依此类推。

索引应小于或等于实际商品数量。调整商品顺序或删除商品后，应重新确认该值指向预期商品。

## 比较行内容块

点击 **添加区块** 可以增加比较行。在左侧列表中拖动内容块可调整前台行顺序，也可以单独隐藏、复制或删除。

### Product Info

显示商品主图和商品标题，通常放在对比表第一行。商品信息来自 **Select Products** 中的商品数据。

建议所有主图采用一致比例和构图。商品标题过长时可能增加整行高度，应在商品后台优化标题并检查移动端显示。

### Price

显示各商品价格。价格格式、货币符号和小数位由 Shopify 市场及商店货币设置决定；促销价格的具体显示取决于商品定价和主题实现。

如果价格不符合预期，请检查商品价格、原价、市场固定价格和当前预览市场，而不是在比较表中手动输入价格。

### Rating

显示商品评分。评分数据通常来自兼容的评论应用或标准商品评分元字段。

若某件商品没有评分，前台可能显示空值或默认状态。发布前应同时测试有评分、无评分和评分数量较少的商品，并确认评分显示与商品详情页一致。

### Ingredients / Metafield

显示成分、材质、规格或其他商品元字段内容。选择该内容块后，根据面板提供的字段设置指定行标题及需要读取的商品元字段。

配置元字段时：

1. 进入 **Shopify 后台 → 设置 → 自定义数据 → 商品**。
2. 创建统一的字段定义，例如材质、容量或适用场景。
3. 为所有参与比较的商品填写相同类型的字段值。
4. 返回主题编辑器，将该比较行连接到对应字段。

未填写字段值的商品可能显示空白或占位符。多个属性需要分别比较时，可按主题允许的方式添加多个 Metafield 行，并为每行使用明确且一致的名称。

### Availability

显示每件商品的库存状态，并使用 **Availability in stock color** 或 **Availability out of stock color** 设置的颜色。

库存状态来自 Shopify 商品及其变体数据。若商品有多个变体，应检查部分变体缺货、全部变体缺货以及允许缺货后继续销售时的显示结果。

## 推荐的行顺序

一组易于理解的排列方式是：

1. Product Info
2. Price
3. Rating
4. Ingredients / Metafield
5. Availability

可以根据顾客决策重点调整顺序。例如，规格是主要差异时可将 Metafield 行提前；库存状态通常适合放在靠后位置。

## Padding

- **Padding Top**：设置分区顶部与上一个分区之间的留白，单位为 `px`。
- **Padding Bottom**：设置分区底部与下一个分区之间的留白，单位为 `px`。

可先让上下间距保持一致，再根据标题行数、比较行数量和相邻分区进行调整。比较表本身内容较多，过大的间距会进一步拉长商品页面。

## 自定义 CSS

展开 **自定义 CSS** 可以为当前分区添加局部样式，例如调整表格边框、列宽或文字换行。

自定义规则应限制在当前分区范围内，避免影响商品页面上的其他表格或商品组件。修改后需测试不同商品数量、长标题、长元字段值和移动端横向浏览；主题升级后也应重新检查兼容性。

## 配置示例

| 设置项 | 示例值 |
| --- | --- |
| Color scheme | 浅色高对比方案 |
| Section heading | `同系列商品对比` |
| Heading size | `2XL` |
| Highlight column background | 浅色强调背景 |
| Availability in stock color | 深绿色 |
| Availability out of stock color | 深红色 |
| Select Products | 4 件同系列商品 |
| Highlight Column Index | `2` |
| 比较行 | 商品信息、价格、评分、材质、库存状态 |
| Padding Top / Bottom | `32 px / 32 px` |

## 发布前检查

1. 确认所选商品均已发布，并在目标市场中可用。
2. 检查商品顺序及 Highlight Column Index，确认突出的是预期商品。
3. 核对商品主图、标题、价格和币种，并测试促销价格状态。
4. 检查评分数据与商品详情页一致，并验证无评分商品的显示。
5. 核对每项商品元字段的名称、值、单位和格式，补齐缺失数据。
6. 测试有库存、缺货、部分变体缺货和允许继续销售等状态。
7. 检查突出列背景、库存颜色、普通文字和边框的对比度。
8. 在桌面端与移动端测试表格横向浏览，确认首列标签和各商品数据容易对应。
9. 使用键盘测试表格中的商品链接，确认焦点状态清晰且目标正确。
10. 检查长商品标题、长元字段值以及不同商品数量下的换行、列宽和行高。
11. 确认自定义 CSS 不会影响页面上的其他分区。
12. 点击 **保存**，再从实际店面完成一次商品比较和进入商品详情页的流程。
