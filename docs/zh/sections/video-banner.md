# 视频横幅（Video Banner）

Video Banner（视频横幅）分区用于展示 Shopify 托管视频或 YouTube、Vimeo 外部视频，并可叠加副标题、主标题和说明文字。它适合品牌故事、产品演示、活动预告和使用场景展示。

![Video Banner 设置面板与前台效果](/images/zh/video-banner.svg)

## 添加分区

1. 进入 **Shopify 后台 → 在线商店 → 主题**。
2. 在当前主题上点击 **自定义**。
3. 打开需要编辑的页面或模板。
4. 点击 **添加分区**，选择 **Video Banner**。
5. 选择 Shopify 托管视频，或填写受支持的外部视频地址。
6. 按需设置移动端视频、封面图、比例和播放方式。
7. 填写叠加文案并调整上下间距。
8. 点击右上角的 **保存**。

此分区不需要添加内容块，视频和文字都在分区设置面板中配置。

## Color scheme

选择当前分区使用的全局配色方案，影响背景、叠加文字、播放按钮及其他主题控件。

点击 **编辑方案** 可进入全局颜色设置。修改方案本身会同时影响其他使用该方案的分区，因此应在视频明暗变化、封面图和加载失败状态下分别检查文字与控件对比度。

## Video

### Shopify-hosted video

选择上传到 Shopify 的桌面端主视频，也可以连接符合条件的动态来源。

使用托管视频时，应先剪除无关片段并压缩文件，在可接受画质下尽量降低体积。上传到 Shopify 文件选择器的媒体会成为店面可访问资源，不应包含机密、个人或敏感内容。

视频上传格式、尺寸和时长限制可能调整，制作素材前应查看 Shopify 最新的文件要求。

### Mobile video

选择专门用于移动端的视频，也可以连接动态来源。该字段为可选项；设置后，移动端会使用它替代桌面端 Shopify 托管视频。

移动端素材建议采用更适合窄屏的构图，并减少时长和文件体积。重要人物、商品和文字应位于安全区域。若留空，移动端会继续使用桌面端视频，并按当前比例和容器方式显示。

### External video URL

填写 YouTube 或 Vimeo 视频地址。只有没有选择 **Shopify-hosted video** 时，外部视频才会显示。

如果同时填写托管视频和外部地址，托管视频优先。使用外部视频时还应确认：

- 视频可公开访问，且没有地区、年龄或嵌入限制。
- 地址来自 YouTube 或 Vimeo，而不是普通网页或其他平台。
- 视频标题、缩略图、字幕和隐私设置正确。
- 第三方播放器及 Cookie 使用符合店铺隐私政策和顾客同意要求。

### 视频来源优先级

分区按以下逻辑选择视频：

1. 移动端且已设置 **Mobile video**：显示移动端视频。
2. 其他情况且已设置 **Shopify-hosted video**：显示 Shopify 托管视频。
3. 未设置 Shopify 托管视频但填写了 **External video URL**：显示 YouTube 或 Vimeo 视频。
4. 没有可用视频：显示封面图或主题提供的空白状态。

更换视频来源后，应重新测试播放控制，因为托管视频和外部播放器支持的行为可能不同。

## Cover image

选择视频加载前或未播放时使用的封面图，也可以连接符合条件的动态来源。

封面图应与视频首帧及内容主题一致，避免顾客点击后看到完全不同的画面。建议使用与 **Aspect ratio** 相同的比例，以减少裁切和页面加载时的布局变化。

即使启用了自动播放，也应设置有代表性的封面图，以应对节省流量模式、自动播放被浏览器阻止、视频加载较慢或播放失败等状态。

## Aspect ratio

设置视频容器的宽高比例：

- **16:9**：适合横向品牌影片、产品演示和大多数桌面视频。
- **4:3**：画面更高，适合较紧凑的横向素材或复古构图。
- **1:1**：正方形比例，适合社交媒体风格内容和移动端展示。

视频或封面图比例与容器不一致时，画面可能被裁切或出现空白区域，具体结果取决于播放器实现。选择后应检查人物、字幕和商品是否完整可见。

## Container width

设置视频横幅的外层宽度：

- **Page width**：视频限制在主题页面容器内，左右保留统一边距。
- **Full width**：视频扩展到可用页面宽度，适合通栏视觉效果。

全宽视频更有视觉冲击力，但需要更高质量的素材。移动端仍应检查页面边缘、圆角和文字安全区域。

## Playback

### Autoplay

控制视频是否在页面加载后自动播放。受浏览器策略、节省流量模式、设备设置和隐私限制影响，自动播放不一定每次都能开始。

自动播放视频必须静音，因此开启该设置时应确保内容在没有声音的情况下仍然能够理解。

### Loop

控制视频播放结束后是否从头继续播放。适合短小、首尾衔接自然的背景片段；较长视频或带有完整叙事的视频通常不需要循环。

### Muted

设置视频初始是否静音。包含重要语音或声音信息时，应提供字幕或文字说明，并让顾客能够主动打开声音。

开启 Autoplay 后，即使这里选择有声状态，主题仍会为了符合浏览器策略以静音方式自动播放。

### Show controls

控制是否显示播放器控制栏。控制栏通常用于播放、暂停、进度和音量等操作，具体项目取决于视频来源和播放器。

如果关闭控制栏，必须确认顾客仍能通过播放按钮、键盘或其他可见控件控制视频。

### Show play button

控制是否显示主题提供的播放按钮。关闭 Autoplay 时建议保留播放按钮，避免顾客无法开始播放。

不要同时关闭 Autoplay、播放器控制栏和播放按钮，否则视频可能没有可发现的启动方式。

### Show mute button（Shopify-hosted video only）

控制是否为 Shopify 托管视频显示独立的静音切换按钮。该设置不适用于 YouTube 或 Vimeo 外部视频。

视频包含音频且初始静音时，建议显示该按钮，并确认图标能够清楚区分静音与有声状态。

## Content

### Subheading

设置主标题上方或附近的简短辅助文字。该字段支持基础富文本格式、链接和动态来源。

可用于填写内容类别、视频时长或简短提示。不需要时可以留空。

### Heading

设置视频横幅的主标题。该字段支持基础富文本格式、链接和动态来源。

标题应概括视频内容，不要只使用与视频无关的促销口号。文案较长时，应检查是否遮挡画面主体或字幕。

### Text

设置补充说明。该字段支持文字样式、加粗、斜体、链接、项目符号和编号列表，并可连接动态来源。

可用一句简短文字说明视频内容、观看价值或无声播放时的关键信息。长段落和复杂列表会遮挡视频，不适合放在横幅叠层中。

### Text alignment

设置 Subheading、Heading 和 Text 的水平对齐方式，可选择左对齐、居中或右对齐。

对齐方式应配合视频主体位置和文字方向。视频左侧有主体时，可将文字放在右侧或居中安全区域；多语言店铺还应检查从右到左语言的排版。

## Padding

- **Padding top**：设置分区顶部与上一个分区之间的留白，单位为 `px`。
- **Padding bottom**：设置分区底部与下一个分区之间的留白，单位为 `px`。

可先让上下间距保持一致，再根据视频比例、容器宽度和相邻分区调整。全宽模式通常不需要过大的额外留白。

## 自定义 CSS

展开 **自定义 CSS** 可以为当前分区添加局部样式，例如限制文字宽度或微调播放器控件位置。

自定义规则应限制在当前分区范围内，避免影响其他视频或媒体组件。修改后需测试所有比例、页面宽度、播放状态以及桌面端和移动端；主题升级后也应重新检查兼容性。

## 无障碍与性能建议

- 为包含对白或重要声音的视频提供准确字幕；有必要时同时提供文字稿。
- 不要只通过音频传递关键信息，自动播放视频应在静音状态下仍可理解。
- 避免快速闪烁、强烈频闪或可能引发不适的画面。
- 顾客必须能够播放、暂停和控制声音，键盘焦点状态应清晰可见。
- 确保字幕不会被标题、说明、播放按钮或其他页面元素遮挡。
- 使用尽可能短的视频并压缩文件，移动端优先提供轻量素材。
- 外部播放器可能增加第三方请求和 Cookie，应结合隐私同意方案测试。
- 封面图应经过压缩，并与视频容器比例保持一致。

## 配置示例

| 设置项 | 示例值 |
| --- | --- |
| Color scheme | 与视频画面有明显对比的方案 |

## 发布前检查

1. 确认视频、封面图和音乐均拥有合法使用权，且不包含敏感信息。
2. 分别测试 Shopify 托管视频、Mobile video 和 External video URL 的来源优先级。
3. 在真实手机上确认 Mobile video 正确替代桌面端视频。
4. 检查 16:9、4:3 和 1:1 比例下的画面裁切、字幕和主体位置。
5. 分别检查 Page width 与 Full width 的页面边缘和画质。
6. 测试 Autoplay 被允许和被阻止时的状态，并确认封面图与播放按钮可用。
7. 测试 Loop、Muted、Show controls、Show play button 和托管视频静音按钮的所有启用状态。
8. 使用键盘操作播放、暂停、进度和静音控件，确认焦点状态清晰可见。
9. 打开字幕并检查其不会被叠加文字或播放器控件遮挡。
10. 检查 Subheading、Heading 和 Text 没有无意义的默认内容，并在不同画面亮度下保持可读。
11. 使用移动网络或限速环境检查封面加载、视频开始时间和页面稳定性。
12. 测试 YouTube 或 Vimeo 视频的公开访问、嵌入权限、隐私同意和失败状态。
13. 确认自定义 CSS 不会影响页面上的其他视频。
14. 点击 **保存**，再从实际店面完成桌面端和移动端播放流程。
