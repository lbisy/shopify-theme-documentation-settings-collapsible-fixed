# Colors

Colors（颜色）设置用于创建和管理商店的配色方案。每套配色方案都可以分别定义背景、文字、边框、输入框、按钮、徽章和反馈消息的颜色，并应用到支持配色方案的页面分区中。

## 打开 Colors 设置

1. 登录 Shopify 后台。
2. 进入 **在线商店 > 主题**。
3. 找到当前使用的主题，点击 **自定义**。
4. 点击左侧的 **主题设置**。
5. 选择 **Colors**。
6. 打开需要修改的配色方案。

![颜色设置](/images/zh/settings/colors.svg)

## 配色方案

配色方案是一组可以重复使用的颜色设置。修改某个方案后，所有正在使用该方案的分区都会同步更新，因此不需要逐个调整页面中的颜色。

你可以为不同的使用场景建立不同方案，例如：

- 浅色方案：适合正文、商品信息和内容较多的区域。
- 深色方案：适合页脚、品牌介绍或需要突出视觉效果的区域。
- 强调色方案：适合促销横幅、订阅区域和行动引导区域。

编辑时，面板顶部的预览区域会显示当前背景、文字和按钮的大致搭配效果。最终效果仍应在桌面端和移动端预览中确认。

::: tip 建议
配色方案不宜过多。保持少量、用途明确的方案，可以让整个商店的视觉风格更加统一，也方便后续维护。
:::

## General colors

General colors（通用颜色）决定当前方案最基础的视觉样式。

- **Background**：设置分区的基础背景颜色。
- **Background gradient**：设置背景渐变。选择渐变后，它会在适用位置取代纯色背景。
- **Text color**：设置主要文字颜色，例如标题、正文和部分图标。
- **Border color**：设置分隔线、卡片边框及其他轮廓元素的颜色。
- **Focus ring color**：设置键盘操作时焦点轮廓的颜色，帮助用户识别当前选中的链接、按钮或输入框。

::: warning 背景渐变
设置 **Background gradient** 后，适用位置将优先显示渐变，而不是 **Background** 中设置的纯色。若希望恢复纯色背景，请清除渐变设置。
:::

背景与文字之间需要保持足够的明暗差异。浅色背景通常搭配深色文字，深色背景通常搭配浅色文字。Focus ring 也应与背景明显区分，避免键盘用户无法判断当前焦点位置。

## Inputs

Inputs（输入框）控制搜索框、联系表单、邮件订阅和其他表单字段的颜色。

- **Input text**：用户输入内容的文字颜色。
- **Input background**：输入框的背景颜色。
- **Input border**：输入框默认状态下的边框颜色。
- **Input placeholder**：占位提示文字的颜色，例如“请输入邮箱地址”。

占位文字应与输入内容有所区别，但不能淡到难以阅读。输入框背景也需要与所在分区的背景保持足够差异，使输入区域清晰可见。

::: tip 检查表单状态
设置完成后，建议实际点击搜索框、联系表单和订阅输入框，检查默认、聚焦和已输入内容等状态是否都清晰可辨。
:::

## Primary button

Primary button（主按钮）用于页面中最重要的操作，例如“添加到购物车”“立即购买”或“提交”。

- **Primary button background color**：主按钮背景颜色。
- **Primary button label color**：主按钮文字及相关图标的颜色。
- **Primary button border color**：主按钮边框颜色。

主按钮应当在页面中足够醒目。按钮文字与背景之间需要保持清晰对比，同时建议让背景色与普通文字链接、次要按钮有所区别。

## Secondary button

Secondary button（次要按钮）用于优先级较低的操作，例如“了解更多”“继续购物”或“返回”。

- **Secondary button background color**：次要按钮背景颜色，也可以设置为透明。
- **Secondary button label color**：次要按钮文字及相关图标的颜色。
- **Secondary button border color**：次要按钮边框颜色。

使用透明背景可以制作描边按钮。此时需要特别检查按钮文字和边框在当前分区背景上是否清晰。如果同一方案可能同时用于纯色和图片背景区域，建议分别预览实际效果。

## Badges

Badges（徽章）用于显示商品状态或提示信息，例如“促销”“新品”或“售罄”。

- **Badge background color**：徽章背景颜色。
- **Badge label color**：徽章文字颜色。
- **Badge border color**：徽章边框颜色。

徽章通常面积较小，因此颜色对比需要更加明确。建议使用简洁的高对比配色，避免使用过于接近的背景色和文字色。

## Feedback messages

Feedback messages（反馈消息）用于向用户显示操作结果或状态提醒。每种消息都包含背景色和前景色；前景色通常应用于文字、图标或边框。

### Success

Success（成功）用于表示操作已经完成，例如商品加入购物车成功或表单提交成功。

- **Success background color**：成功消息背景颜色。
- **Success foreground color**：成功消息文字和图标颜色。

### Warning

Warning（警告）用于提醒用户注意可能存在的问题或需要确认的信息。

- **Warning background color**：警告消息背景颜色。
- **Warning foreground color**：警告消息文字和图标颜色。

### Error

Error（错误）用于显示操作失败、字段填写错误或其他需要立即处理的问题。

- **Error background color**：错误消息背景颜色。
- **Error foreground color**：错误消息文字和图标颜色。

### Info

Info（信息）用于显示一般说明、补充提示或非紧急状态信息。

- **Info background color**：信息消息背景颜色。
- **Info foreground color**：信息消息文字和图标颜色。

::: info 可访问性提示
不要只依靠颜色表达成功、警告或错误状态。保留主题提供的文字说明和状态图标，可以帮助色觉差异用户正确理解消息。
:::

## 删除配色方案

面板底部的 **移除配色方案** 用于删除当前方案。删除前，请确认页面分区是否仍在使用该方案，并为相关分区选择其他合适的配色方案。

::: warning 注意
删除操作可能无法直接恢复。建议先记录重要颜色值，或确认不再需要该方案后再进行删除。
:::

## 保存与检查

完成颜色设置后，点击主题编辑器右上角的 **保存**。建议检查以下内容：

- 标题、正文和链接在背景上是否清晰。
- 主按钮是否醒目，次要按钮是否仍容易识别。
- 输入框的文字、占位文字和边框是否清楚。
- 商品徽章在不同商品图片附近是否容易阅读。
- 成功、警告、错误和信息消息是否容易区分。
- 键盘操作时的焦点轮廓是否可见。
- 同一配色方案在桌面端和移动端是否保持良好效果。
- 使用背景渐变的区域是否仍能保证文字和按钮的可读性。

颜色面板中的十六进制值（例如 `#1B2900`）可以直接复制和填写，方便在不同方案之间保持一致的品牌颜色。
