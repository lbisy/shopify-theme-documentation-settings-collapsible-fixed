# Cart

Cart（购物车）设置用于决定顾客查看购物车时的展示方式。此设置会影响全站的购物车入口，以及商品加入购物车后的主要交互体验。

## 打开 Cart 设置

1. 登录 Shopify 后台。
2. 进入 **在线商店 > 主题**。
3. 找到当前使用的主题，点击 **自定义**。
4. 点击左侧的 **主题设置**。
5. 选择 **Cart**。

![Cart 设置](/images/zh/settings/cart.svg)

## Cart type

**Cart type** 用于选择购物车以侧边抽屉还是独立页面的形式显示。

### Drawer

选择 **Drawer** 后，购物车会以抽屉面板的形式在当前页面打开。顾客不需要离开正在浏览的页面，就可以查看或修改购物车。

Drawer 模式通常具有以下特点：

- 加入商品后可以快速查看购物车内容。
- 顾客能够继续停留在当前商品页或商品列表页。
- 适合希望缩短购物流程、鼓励顾客继续浏览商品的商店。
- 在桌面端通常从页面侧边打开，在移动端则会适配较窄的屏幕。

::: tip 适用场景
如果商店商品购买决策较快，或者希望顾客连续添加多个商品，建议使用 Drawer。
:::

### Page

选择 **Page** 后，顾客打开购物车时会进入独立的购物车页面。

Page 模式通常具有以下特点：

- 为购物车商品、数量调整和订单信息提供更完整的页面空间。
- 适合购物车内容较多，或需要顾客仔细确认商品信息的商店。
- 页面地址更加明确，方便顾客刷新、返回或分享当前购物车页面入口。
- 在桌面端和移动端都采用完整页面布局。

::: tip 适用场景
如果商品需要较长的说明、购物车中经常包含多个商品，或者结账前需要展示更多提示信息，建议使用 Page。
:::

## 如何选择

可以根据商店的购买流程进行选择：

- 希望顾客快速加购并继续浏览：选择 **Drawer**。
- 希望顾客集中确认购物车内容：选择 **Page**。
- 商品数量较少、购买流程简单：优先考虑 **Drawer**。
- 购物车信息较复杂、需要更多显示空间：优先考虑 **Page**。

无论选择哪一种方式，顾客的购物车商品数据都不会改变；此设置只会改变购物车的展示和访问方式。

::: info 提示
“立即购买”等动态结账按钮可能会直接进入结账流程，不一定先打开购物车。Cart type 主要影响普通“添加到购物车”及购物车图标的交互。
:::

## 保存与检查

完成设置后，点击主题编辑器右上角的 **保存**。建议分别在桌面端和移动端测试以下操作：

1. 从商品页面将商品加入购物车。
2. 点击页眉中的购物车图标。
3. 修改商品数量。
4. 删除购物车中的商品。
5. 从购物车进入结账页面。
6. 返回商店并继续添加其他商品。

确认购物车能够按照所选类型正常打开，商品价格、数量和结账按钮也能完整显示。
